## The setup

MCP servers fail in three places: **negotiation**, **schema**, and **business logic**. Test all three, in that order, cheapest first.

## Picture this

<Diagram id="test-harness-topo" caption="Test runner drives in-memory or fixture transport into server under test." alt="Diagram test harness transport server" />

## Mental model

Treat your server like a library with a **wire protocol façade**—unit test the core, contract test the façade.

## Walkthrough

- Snapshot stable JSON outputs—not full timestamps unless frozen.
- Add regression tests for **bad model inputs** (they will happen).
- Build a tiny “fake client” script for reproducing bugs.

## Hall of Shame

<HallOfShame tag="observability" title="printf debugging on stdout">

## Why this matters in production

Incidents love **non-reproducible** bugs. Protocol-level tests turn “Claude weirdness” into failing CI.

## Mini challenge

Write one test that asserts a tool rejects a path traversal payload.

## Reflection

What’s the smallest harness that would have caught your last bug in 10 minutes?
