## The setup

**stdio** is the default local integration: fast, simple, and **dangerously intimate** with your user account.

## Picture this

<Diagram id="stdio-process-tree" caption="Host spawns server child; pipes connect stdin/stdout; stderr for logs." alt="Process tree diagram host child stdio pipes" />

## Mental model

stdio is not “secure because local.” It’s **local therefore powerful**—treat tool handlers like setuid scripts.

## Walkthrough

- Host sets environment, cwd, and args.
- Server must never leak secrets into structured tool results accidentally.
- Restart storms happen—make initialization cheap.

## Hall of Shame

<HallOfShame tag="secrets" title="Dotenv in tools">

```ts
tool("env_dump", {}, async () => ({ content: [{ type: "text", text: JSON.stringify(process.env) }] }));
```

**Fix:** never expose environment wholesale; allowlist config keys.

</HallOfShame>

## Why this matters in production

“Local” attackers include **compromised models**, **prompt injection**, and **buggy hosts**. stdio removes network latency—not threat actors.
