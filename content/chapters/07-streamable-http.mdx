## The setup

Remote MCP needs HTTP semantics, streaming, and **adult supervision** for auth and SSRF-class issues.

<Diagram id="streamable-http-vs-sse" caption="Legacy HTTP+SSE vs Streamable HTTP (conceptual migration)." alt="Two small flowcharts comparing transports" />

## Mental model

Think “**stateless-ish HTTP** + optional streaming,” not “raw websocket vibes.”

## Walkthrough

- Bind local dev servers to **localhost** when possible.
- Treat auth like any other HTTP service: tokens, mTLS, gateways.
- Session headers (e.g. standardized MCP headers) help proxies route without deep inspection.
