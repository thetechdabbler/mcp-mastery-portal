## The setup

MCP rides **JSON-RPC**. If you treat messages like REST paths, you’ll mis-handle notifications and lifecycle edge cases.

## Picture this

<Diagram id="json-rpc-anatomy" caption="Request/response vs notification shapes (conceptual)." alt="Diagram three boxes request response notification" />

## Mental model

**Initialize** is where both sides show their homework. After that, steady-state traffic is cheaper—but **reconnect** paths must be idempotent.

## Walkthrough

- Requests carry `id` and expect `result` or `error`.
- Notifications never block the caller waiting for a response.
- Lifecycle restarts happen more than you’d like—design handlers to be safe on duplicate init.

## Hall of Shame

<HallOfShame tag="lifecycle" title="Skip initialize">

## Why this matters in production

Version skew between hosts and servers is normal. Lifecycle discipline is how you avoid **silent partial feature** bugs.
