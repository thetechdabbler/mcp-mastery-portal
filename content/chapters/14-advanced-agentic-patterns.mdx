## The setup

Agentic workflows are **graphs** of decisions. MCP servers are **nodes**. If you don’t label edges with trust, the graph eats you.

## Picture this

<Diagram id="multi-server-workflow" caption="Host orchestrates multiple servers with trust tiers." alt="Multi server orchestration diagram" />

## Mental model

Treat orchestration like **service mesh routing**: identities, policies, budgets, and kill switches.

## Walkthrough

- Keep “planner tools” separate from “executor tools” where possible.
- Cap loops: max steps, max wall time, max spend.
- Prefer explicit user intent for high-risk transitions.

## Hall of Shame

<HallOfShame tag="agents" title="Unbounded loops">

```txt
while true: call_tools_until_happy()
```

**Fix:** budgets, checkpoints, and deterministic stop conditions.

</HallOfShame>

## Why this matters in production

Agents fail open. Production systems fail **closed**.

## Mini challenge

Design a two-tier tool system: “planner-safe” vs “needs ticket id”.
