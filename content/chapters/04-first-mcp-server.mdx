## The setup

You’ll use `McpServer` from `@modelcontextprotocol/sdk/server/mcp.js` and `StdioServerTransport`—the same primitives the labs compile against.

## Picture this

<Diagram id="first-server-sequence" caption="stdio MCP: stdin JSON-RPC in, stdout JSON-RPC out, handler executes tool." alt="Sequence diagram host client server stdio" />

## Mental model

Your server is a **JSON-RPC worker** with superpowers: advertise tools, validate inputs, return structured content.

## Walkthrough

1. Create `McpServer` with name/version.
2. `registerTool` with `inputSchema` as Zod shapes.
3. `connect(new StdioServerTransport())`.
4. Log to **stderr** only.

## Hall of Shame

<HallOfShame tag="transport" title="stdout logging">

```ts
console.log("debug", payload);
```

**Fix:** `console.error` or structured logging to stderr; keep stdout pristine.

</HallOfShame>

## Why this matters in production

stdio servers inherit the host’s user—**ambient authority** is real. Tiny bugs become exfiltration because the model is an adversarial caller.

## Mini challenge

Add a tool that **cannot** succeed unless schema validation rejects bad types—prove it with a manual JSON-RPC call.
